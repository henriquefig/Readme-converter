Installation

Install Xampp (or equivalent) in your device of choice.

You'll need a special configuration for Apache. In the Xampp control panel open this folder:

Inside that folder, open the following file:

conf/extra/httpd-vhosts.conf

Add these lines to the file:

<VirtualHost *:80>
    DocumentRoot "C:\Path_to_your_workspace"
    ServerName localhost
    ServerAlias *.localhost
    <Directory "C:\Path_to_your_workspace">
    </Directory>
</VirtualHost>

Clone the repo to your desired path through the command (this must be done through a Linux terminal or Git Bash):

git clone ssh://altecis@altec-is.pt/home/altecis/git.repo/hrm_dev

In your OS of choice, unhide hidden folders:

  • Windows 10:

    • In the search box on the taskbar, type folder, and then select Show hidden files and folders from the search results.
    • Under Advanced settings, select Show hidden files, folders, and drives, and then select OK.

  • Mac OS:

    • Open a folder and press CMD + . to toggle the hide/unhide of the folders/files

  • Linux:

    • Depends on the distro being used... (ls -a will list the hidden files/foldres on the terminal though)

Open the folder where you've installed the repo and find the folder .git. Inside it you will look for the file .git/info/exclude. Open it with your text editor of choice.

Note: Even though exclude file has no extension, it can be opened as a simple text file.

Add the following lines to the exclude file:

*.bak.php
*.bak.ini
*.bak.htaccess
files/
temp/
logs/
files_default/

Save the file and your repo is ready!

Now you need to create the clients management database.

In your repo root directory, you'll find a file called init.sql.

Open a MySQL console (in Xampp just click shell) login with your root user and type:

CREATE DATABASE hrm_signup;
quit;

After that type:

mysql -u root -h localhost -D hrm_signup < path_to_the_repo/init.sql

Note: If you've set a password for your root user, add -p your_password to the command

Congratulations your HRM installation for local development is now complete!

You can create your test first client in the Signup Page

Contributing

When contributing to this repo please make sure you do not edit your .git/info/exclude! When making commits, please make sure you use the command in this form:

git commit -am "Brief summary of changes"

Remember that when merging to the server version, replace the excluded files with corresponding .bak files.

Special development notes

Please remember to setup the SMTP server to Mailtrap to avoid email spamming.

The emails recieved through this application will have incorrect links for local development. To correct them make the following changes on a link.

Example link:

https://domain.altec-is.pt/backoffice

Corrected link:

http://domain.localhost/repo_name/backoffice
